# Docs Journal Rules

This `docs/` tree is project memory for humans and coding agents.

## Core rules

- Treat `docs/progress` as the active append-only journal for repository memory.
- Legacy docs under `docs/decisions`, `docs/issues`, and `docs/principles` remain append-only historical records if you need to reference them.
- Never delete, rewrite, reorder, or "clean up" old entries.
- To correct something, append a correction that references the earlier doc and timestamp.
- Keep entries short, factual, and link-rich. Avoid raw terminal spam.
- Use ISO 8601 UTC timestamps.
- When a local `AGENTS.md` exists in a subfolder, follow the more specific rule set there.
