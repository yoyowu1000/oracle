# Progress Journal Rules

- This folder is append-only.
- Do not delete, rewrite, reorder, or summarize away old history.
- Add new material only by appending to the end of files.
- If an older entry is wrong, append a correction instead of editing the old entry.
- Index files are chronological append-only logs; new rows go at the end.

- Record only meaningful milestones: start of non-trivial work, change of approach, successful validation, blocker, or handoff.
- Prefer one progress file per feature, area, or task stream.
- Include touched files, validation steps, and links to related docs when relevant.
